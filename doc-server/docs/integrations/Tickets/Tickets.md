# Tickets

NudgeBee integrates with popular ticketing and incident management systems so you can create, track, and manage tickets directly from the NudgeBee platform. Tickets can be created manually from the UI for errors, events, recommendations, and logs — or automated using [Autopilot](../../features/autopilot/autopilot.md) to create and assign tickets based on pre-configured rules.

### Why Connect a Ticketing System?

Connecting a ticketing system allows NudgeBee to:

- **Create tickets automatically** when issues are detected (via Autopilot rules).
- **Search for similar past tickets** to help diagnose recurring problems.
- **Link incidents to existing tickets** for better tracking and context.
- **Read and respond to tickets** with AI-generated insights and recommended actions.
- **Reference runbooks** attached to tickets for faster resolution.

:::tip
If your team uses a ticketing system as the primary incident workflow, connecting it to NudgeBee ensures that alerts and recommendations flow directly into your existing process — no context switching needed.
:::

### When Do You Need This?

This integration is **optional**. NudgeBee works fully without a ticketing system. Connect one when you want to:

- Automate incident ticket creation from NudgeBee events.
- Let NudgeBee research historical tickets for similar issues.
- Centralize incident management alongside NudgeBee's troubleshooting capabilities.

## Available Ticket Integrations

*   **[Jira](./jira.md)** — Create and manage Jira issues from NudgeBee.
*   **[GitHub Issues](./github_issues.md)** — Create GitHub Issues for tracked repositories.
*   **[GitLab Issues](./gitlab.md)** — Create GitLab Issues for tracked projects.
*   **[ServiceNow](./servicenow.md)** — Integrate with ServiceNow for enterprise ITSM workflows.
*   **[PagerDuty](./pagerduty.md)** — Trigger and manage PagerDuty incidents.
*   **[Zenduty](./zenduty.md)** — Integrate with Zenduty for incident management.
