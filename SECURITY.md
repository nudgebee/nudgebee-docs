# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in NudgeBee or
its documentation, please report it privately. **Do not open a public
GitHub issue, discussion, or pull request.**

Email: **security@nudgebee.com**

Please include:

- A description of the issue and the impact
- Steps to reproduce, or a proof-of-concept if available
- The affected version or commit
- Any suggested mitigations

## What to Expect

- We will acknowledge your report within **3 business days**.
- We will provide an initial assessment and an expected timeline
  within **10 business days**.
- We will keep you informed of progress through to remediation.
- If the issue is confirmed, we will credit you in the release notes
  unless you prefer to remain anonymous.

## Coordinated Disclosure

We ask that you give us a reasonable window to investigate and ship
a fix before disclosing the issue publicly. We will work with you on
a disclosure timeline and aim to have a fix or mitigation in place
within **90 days** of report, sooner for high-severity issues.

## Scope

This policy covers the NudgeBee documentation repository
(`nudgebee/nudgebee-docs`) and the published documentation site at
`docs.nudgebee.com`. For vulnerabilities in NudgeBee product code,
agents, server, or hosted services, the same email address applies.

## Out of Scope

- Issues in third-party dependencies that have not been triaged by
  upstream — please report those to the upstream project first.
- Findings from automated scanners without a demonstrated impact.
- Best-practice recommendations that are not exploitable.
